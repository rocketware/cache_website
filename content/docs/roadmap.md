# Roadmap

Cache is actively developed. Here's where we're heading.

## In Progress

**Version 1.0** — The initial public release. Available on macOS, Linux, iOS, and Android. Core functionality is stable; remaining work focuses on polish and resolving any outstanding issues.

See the [Changelog](/docs/changelog/) for current progress.

## Coming Soon

### Frontmatter Data Fields

Attach structured data to notes via frontmatter—phone numbers, addresses, dates, tags, or any custom fields. This data will surface in the note sidebar as rich, formatted content.

Use cases: contact details on People notes, project metadata, recipe ingredients, or whatever structure fits your workflow. Future iterations may expose this data in directory views or custom list templates.

### Web Clipper

Save content from the web for later. The clipper will parse pages into clean markdown and store them as notes in your repository. No cloud service, no proprietary format—just markdown files you own.

### AI Integration

Don't panic. Cache is, by design, a simple tool. It will stay that way.

That said, there are genuine use cases where AI can help without compromising the app's principles:

- **Your terms**: AI features will be entirely optional
- **Your models**: Configure your own providers—major services or self-hosted models
- **Your data**: You control exactly what the models see

Like Git integration, AI will be transparent infrastructure, not a black box making decisions for you.
