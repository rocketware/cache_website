Cache v0.3.3

April 02, 2026

← Releases

Changelog

  • Add support for multiple git remotes with configurable default — set preferred remote from the remotes settings tab
  • Google Play Edition: add in-app purchase support for seamless upgrades
  • Add restore purchases option for app store editions
  • Improve license management — redesigned repository slots, actionable status notifications, and automatic verification with 4-week offline grace period
  • Improve post-purchase experience when no git remote is configured
  • Improve git remote and credential dialogs — added setup guide links, smarter defaults, and snackbar validation errors
  • Improve git remote status UX: error badges, configure action on license page, and warnings when no remote is set
  • Improve git sync error reporting — network, authentication, and DNS failures now correctly identified and surfaced
  • Improve error diagnostics — git sync, license verification, and purchase flows now surface clear, actionable messages
  • Improve responsiveness of file and folder operations — changes now reflected immediately
  • Improve directory configure dialog layout and usability
  • Improve privacy by disabling Google Play Billing's internal telemetry
  • Add warning dialog when opening Cache system files with option to configure or open directly
  • Add notice that git credentials are shared across all repositories on the device
  • Fix directory listing showing config name instead of folder name
  • Fix git remote changes not reflected on license and directory screens until app restart
  • Fix remote sync status not displaying correctly at app startup
  • Fix expired license briefly showing as valid
  • Fix duplicate purchase notification snackbars
  • Fix error when resizing window with settings dialog open
