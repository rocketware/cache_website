Build Verification

Cache direct downloads are cryptographically signed so you can verify integrity and authenticity before installation.

Signed manifest

Every release includes a signed manifest (e.g. v0.7.0.manifest.toml) listing all artifacts with SHA-256 checksums. Verifying the manifest signature proves both integrity and authenticity.

Public key

RWSuyCCt9+/8XP0AK3jidFQotJmj82u3RQvmTRCHZeW460xcSsjxH8RQ

Download: cache-minisign.pub (also on GitHub).

Verification steps

1. Install Minisign

2. Download the manifest and signature

From any release page, download .manifest.toml and .manifest.toml.minisig. Available from three independent sources:

  • Website: https://www.cachenotes.app/downloads/releases/v0.7.0.manifest.toml
  • GitHub: release assets on github.com/rocketware/cache/releases
  • GCP mirror: https://download.rocketware.io/cache/v0.7.0/

3. Verify the signature

minisign -Vm v0.7.0.manifest.toml -p cache-minisign.pub

If valid, you'll see: Signature and comment signature verified.

4. Compare SHA-256 checksum

shasum -a 256 your-downloaded-file

Match the output against the checksum_sha256 field in the manifest for that artifact.

Android APK

Verify the signing certificate fingerprint to confirm the APK is an authentic Cache build.

Package IDs: io.rocketware.cache (Direct/Accrescent), io.rocketware.cache.googleplay (Google Play)

Certificate fingerprint (SHA-256)

E0:FB:26:D7:60:32:37:72:B2:47:B4:D3:5D:34:2F:B0:EF:40:9B:17:22:7C:DB:CA:A8:43:D3:9A:24:FA:43:1E

On device: Install AppVerifier and compare against the fingerprint above.

On desktop:

apksigner verify --print-certs cache.apk
Compare the Signer #1 certificate SHA-256 digest against the fingerprint above.

macOS

Verify the DMG checksum against the signed manifest:

shasum -a 256 cache-0.7.0-macos-universal.dmg

After installing, you can verify the code signature and Team ID:

codesign --verify --deep /Applications/Cache.app
codesign -dv /Applications/Cache.app 2>&1 | grep TeamIdentifier

Expected Team ID: U4KM65Y22D (Rocketware Limited)

Linux

Linux direct downloads are verified via the signed manifest. Download your artifact, then follow the manifest verification steps to confirm the SHA-256 checksum.
