Build Verification

All Cache direct downloads are cryptographically signed using Minisign, a simple and lightweight tool for verifying file integrity and authenticity.

What is signed

Each release includes a signed manifest (e.g. vX.Y.Z.toml) listing every artifact with its SHA-256 checksum. The manifest is signed with Cache's Ed25519 key — verifying the signature proves both integrity and authenticity of all listed artifacts.

Public key

RWSuyCCt9+/8XP0AK3jidFQotJmj82u3RQvmTRCHZeW460xcSsjxH8RQ

Download: cache-minisign.pub (also available on Github).

Verification steps

1. Install minisign

Install Minisign for your platform.

2. Download the manifest and signature

From any release page, download the .toml manifest and its .toml.minisig signature. These are available from three independent sources:

  • Website: https://www.cachenotes.app/releases/vX.Y.Z.toml
  • GitHub: release assets on github.com/rocketware/cache/releases
  • GCP mirror: https://download.rocketware.io/cache/vX.Y.Z/

3. Verify the signature

minisign -Vm vX.Y.Z.toml -p cache-minisign.pub

If valid, you'll see: Signature and comment signature verified.

4. Compare SHA-256

After verifying the manifest, compare the checksum of your downloaded artifact:

sha256sum cache-X.Y.Z-linux-x64.AppImage

Match the output against the sha256 field in the manifest for that artifact.

Platform app stores

Downloads from the Apple App Store, Google Play, and Microsoft Store are signed through each platform's own trust chain. Minisign verification applies to direct downloads (AppImage, deb, rpm, tar.gz) distributed outside of app stores.
